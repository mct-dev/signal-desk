### 1. AI Agent Operations and Recovery

- Score: 90
- Evidence: 50 current items across 7 source types
- Customer: teams using browser or workflow agents in production

- Wedge: operations layer for agent runs with checkpoints, human handoff, replay, and failure taxonomies
- Why now: recent public seeds around orchestration tax, Appshots context capture, agent benchmark failure, and SQLite's AGENTS.md policy all point toward agent workflows needing supervision and governance rather than more raw autonomy
- Validation: interview teams running agents against real browsers, support queues, internal workflows, or coding-agent pipelines

### 2. LLM Evaluation Regression Workflows

- Score: 90
- Evidence: 33 current items across 7 source types
- Customer: product engineering teams shipping LLM features
- Pain: teams need repeatable ways to catch quality regressions before LLM features reach customers
- Wedge: CI-linked eval runner that turns production failures and agent failures into regression suites with plain-language reports
- Why now: the Mitchell Hashimoto benchmark-gaming seed adds a clean public example of tests or metrics improving while product quality gets worse
- Validation: collect 15 examples of teams reverting prompts, models, or agent strategies because tests missed a regression

### 3. Usage-Based Billing Operations

- Score: 90

- Why now: this remains the densest buyer-pain cluster, with recurring pain and spend language around credits, invoice reconciliation, and usage tiers
- Validation: talk to 10 API startups that introduced credits or usage tiers in the last year

## General Market Trends for Future Market-Trends Page

- Agent work is becoming an operations problem: teams need supervision, replay, human handoff, context capture, and review queues.
- Agent governance is moving into developer workflow: contribution policies, benchmark-gaming examples, MCP/tool config concerns, and app-context capture are converging.
- Evaluation needs product judgment: teams want regression evidence that catches broken user outcomes, not only better benchmark numbers.
- Usage pricing keeps creating back-office pain: AI/API companies need reconciliation, credit, dispute, and renewal support as pricing gets more granular.
- Enterprise-readiness work is shifting earlier: SSO, SOC2, privacy review, and procurement evidence are increasingly startup-stage problems.
